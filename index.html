Smart Kos IoT System

6-Room Modular Building Automation Proposal

System Architecture

System Overview

The proposed solution uses a distributed modular architecture. Each room has its own controller module (Slave Node) connected through an RS485 Modbus RTU network to a central Master Controller. The Master Controller communicates with the MQTT server and dashboard through LAN/VLAN.

Hardware Architecture

Slave Node (Per Room)

  • 4x Relay Outputs
  • IR AC Controller
  • Door Sensor
  • Motion Sensor
  • PZEM Energy Monitor
  • Local Switch Inputs
  • RS485 Modbus RTU

Master Controller

  • ESP32-S3
  • Ethernet LAN/VLAN
  • MQTT Client
  • RS485 Master
  • Automation Engine
  • Dashboard Integration
  • Future OTA Capability

Supported Microcontrollers for Slave Node

Microcontroller Suitable Notes
ESP32 ✅ WiFi capable, large memory
ESP32-S3 ✅ Higher performance and RAM
STM32 ✅ Industrial-grade reliability
Arduino Nano (ATmega328P) ✅ Fully capable as Modbus RTU slave
RP2040 / PIC / Others ✅ As long as UART Modbus RTU is supported

Communication Specifications

  • Protocol : Modbus RTU
  • Physical Layer : RS485 (2-wire half duplex)
  • Topology : Daisy Chain
  • Cable : CAT5e / CAT6 (RJ45)
  • Baudrate : 9600 – 115200 bps
  • Maximum Nodes : 247
