System Architecture

System Overview

The proposed solution uses a distributed modular architecture. Each room has its own controller module (Slave Node) connected through an RS485 Modbus RTU network to a central Master Controller. The Master Controller communicates with the MQTT server and dashboard through LAN/VLAN.

Hardware Architecture

Slave Node (Per Room)

  • 4x Relay Outputs
  • IR AC Controller
  • Door Sensor
  • Motion Sensor
  • PZEM Energy Monitor
  • Local Switch Inputs
  • RS485 Modbus RTU

Master Controller

  • ESP32-S3
  • Ethernet LAN/VLAN
  • MQTT Client
  • RS485 Master
  • Automation Engine
  • Dashboard Integration
  • Future OTA Capability

Recommended Slave Node Platform

Why ESP32 is Recommended

  • Large Flash and RAM for future feature expansion.
  • Built-in WiFi and Bluetooth for commissioning and maintenance.
  • Supports Over-The-Air (OTA) firmware updates.
  • Multiple UART, SPI, I²C, PWM, and ADC peripherals.
  • Sufficient processing power for Modbus RTU, sensor monitoring, and local automation simultaneously.
  • Large ecosystem and long-term availability.
  • Cost-effective compared to industrial PLC hardware.
  • Single hardware platform simplifies manufacturing and maintenance.

Recommended Master Node Platform

Why ESP32-S3 is Recommended for the Master Controller

  • Higher processing performance suitable for managing multiple room nodes simultaneously.
  • Larger RAM and Flash capacity for MQTT, automation rules, and future software expansion.
  • Native support for secure networking and OTA firmware updates.
  • Supports Ethernet interfaces (LAN8720/W5500) for reliable LAN/VLAN connectivity.
  • Sufficient resources for local dashboards, logging, and data buffering.
  • Dual-core architecture allows separation of communication and application tasks.
  • Large ecosystem and long-term support reduce development and maintenance costs.
  • Provides headroom for future features without requiring hardware redesign.

Communication Specifications

  • Protocol : Modbus RTU
  • Physical Layer : RS485 (2-wire half duplex)
  • Topology : Daisy Chain
  • Cable : CAT5e / CAT6 (RJ45)
  • Baudrate : 9600 – 115200 bps
  • Maximum Nodes : 247
